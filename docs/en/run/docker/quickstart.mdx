## **Prerequisites**

- Linux ARM64 host with Docker installed.
- Qualcomm IoT device — see [Supported devices](/en/get-started/devices#iot-linux-arm64).
- GitHub Personal Access Token (PAT) with read access to GHCR.

## **Pull the image**

Authenticate to GitHub Container Registry, then pull:

```bash bash
export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
export GITHUB_USERNAME=your_github_username
docker login ghcr.io -u "$GITHUB_USERNAME" -p "$GITHUB_TOKEN"
```

You should see a "Login Succeeded" message. Then pull the latest image:

```bash bash
docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
```

## **Run your first model**

Drop into an interactive shell inside the container on a local Linux ARM64 machine or a [QDC device via SSH](#connecting-to-a-qdc-device):

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  ghcr.io/qcom-ai-hub/geniex-cli:latest
```

Then run a model from the container shell:

```bash bash
geniex infer Qwen/Qwen3-0.6B-GGUF
```

<Note>
The `--privileged` flag is required for NPU/GPU device access. Replace `Qwen/Qwen3-0.6B-GGUF` with any supported model — see [Supported models](/en/models/supported).
</Note>

## **Connecting to a QDC device**

To test on a Qualcomm Device Cloud (QDC) device, follow these steps to establish an SSH connection:

### **Step 1: Setup SSH tunneling**

In the QDC portal, navigate to **Interactive Sessions** and click **Connect** to view the SSH connection instructions.

<img
  src="/Images/qdc/qdc-1.png"
  alt="QDC portal showing the Connect button for SSH instructions"
  style={{ borderRadius: '0.5rem' }}
/>

Open a terminal and run the SSH tunnel command provided by QDC. Replace `<PRIVATE_KEY_FILE_PATH>` with the path to your private key file:

<img
  src="/Images/qdc/qdc-2.png"
  alt="SSH tunneling setup with private key and port forwarding"
  style={{ borderRadius: '0.5rem' }}
/>

### **Step 2: Connect to the device**

Open a new terminal and SSH into the device using the forwarded port. The default password is `oelinux123`:

```bash bash
ssh -o StrictHostKeychecking=no -o UserKnownHostsFile=/dev/null -p 2222 root@localhost
```

<img
  src="/Images/qdc/qdc-3.png"
  alt="Successful SSH connection to the QDC device"
  style={{ borderRadius: '0.5rem' }}
/>

Once connected, you can run the Docker commands above directly on the device.
