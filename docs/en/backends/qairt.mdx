The `qairt` backend runs pre-compiled bundles from **Qualcomm AI Hub** through the Qualcomm AI Runtime. It's NPU-only, with the bundle compiled and quantized for a specific Snapdragon chipset — typically the fastest NPU path when your model is available on AI Hub.

## **Devices**

QAIRT is **NPU only**. The device alias maps as:

| Device alias | Effect |
|---|---|
| `npu` *(default)* | Pin `HTP0` — the only supported path. |
| `cpu` / `gpu` | Coerced to `npu` with a warning. Never an error. |

If you don't pass `--device`, `qairt` defaults to `npu`.

## **Quantization & context**

QAIRT bundles are **pre-quantized at compile time**. The KV cache size and context length are also baked into the bundle — neither can be changed at runtime.

In Android, this means `nGpuLayers != 0` and `nCtx != 0` are rejected with `PARAM_NOT_SUPPORTED`. Leave both at defaults and tune `max_tokens` / `enable_thinking` only.

## **Picking a model**

QAIRT models follow two patterns:

**1. AI Hub `qualcomm/*` repos** — pull and run directly:

```powershell windows
geniex infer qualcomm/Qwen3-4B
```

**2. Self-converted bundles** — download locally first, then pull with `--model-hub localfs`. See [Bring your own model](/en/models/bring-your-own#qairt-self-converted-bundles).

See [Supported models](/en/models/supported#qairt-ai-hub-bundles) for the full list.

## **Chipset on Android**

AI Hub pulls on Android **require an explicit chipset** — auto-detect only runs on Windows on Snapdragon. Use:

| Chipset | Phone |
|---|---|
| `SM8750` | Snapdragon 8 Elite |
| `SM8850` | Snapdragon 8 Elite Gen 5 |

See the [Android API reference](/en/run/android/api-reference#model-manager) for `ModelPullInput.chipset`.

## **Limitations**

- NPU only — no CPU/GPU fallback.
- Context length and KV cache are fixed by the bundle.
- Adding a new AI Hub model requires registering it on the C++ side first (`third-party/geniex-qairt/models/{llm,vlm}_model_registry.h`).
