There are two ways to get the GenieX Android demo running: install the prebuilt APK (fastest) or build the demo app from source in Android Studio (lets you modify the demo UI and pin a specific SDK version from Maven Central).

## **Option 1: Install the prebuilt APK**

Download the latest demo APK:

<Card
  title="geniex-demo.apk"
  icon="download"
  href="https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-demo.apk"
>
  Latest GenieX Android demo APK. Signed debug build — install with `adb install` or upload to a QDC session.
</Card>

### **1a. With a physical Snapdragon 8 Elite device**

Requires [`adb`](https://developer.android.com/tools/adb) on your host machine and a phone with **Snapdragon 8 Elite** (SM8750) or **Snapdragon 8 Elite Gen 5** (SM8850).

<Steps>
  <Step title="Enable USB debugging">
    On the phone, open **Settings → About phone**, tap **Build number** seven times, then go to **Settings → System → Developer options** and enable **USB debugging**.
  </Step>
  <Step title="Connect and verify">
    ```bash
    adb devices
    ```
    The phone should appear with state `device`. If it shows `unauthorized`, accept the RSA prompt on the phone.
  </Step>
  <Step title="Install the APK">
    ```bash
    adb install -r geniex-demo.apk
    ```
    `-r` lets you reinstall over a previous version without uninstalling first.
  </Step>
  <Step title="Launch">
    Open **GenieX Demo** from the app drawer. Models download from Hugging Face / Qualcomm AI Hub on first use — keep the phone on Wi-Fi.
  </Step>
</Steps>

### **1b. Without a physical device — use Qualcomm Device Cloud (QDC)**

If you don't have a Snapdragon 8 Elite or 8 Elite Gen 5 phone on hand, you can rent one by the minute on Qualcomm Device Cloud.

<Steps>
  <Step title="Pick a device">
    Sign in to [Qualcomm Device Cloud](https://qdc.qualcomm.com/), choose **Snapdragon 8 Elite** or **Snapdragon 8 Elite Gen 5**, and select **Interactive session**.
  </Step>
  <Step title="Configure the session">
    Before starting the session, enable:
    - **Wi-Fi** — required so the demo can download models from Hugging Face / AI Hub.
    - **Keep screen on** — avoids the screen locking mid-download.

    SSH is **not** required for this flow.
  </Step>
  <Step title="Upload the APK">
    **Before starting the session**, use **Upload file** in the session setup page to push `geniex-demo.apk` onto the device. The file must be uploaded before the session begins.
  </Step>
  <Step title="Start and test">
    Start the session. In the mirrored screen, tap the uploaded APK to install it, then open **GenieX Demo** from the app drawer — no local adb needed.
  </Step>
  <Step title="(Optional) Test a VLM with an image">
    To try a vision-language model, upload a test image via the QDC terminal:
    ```bash
    curl -L "https://s7d1.scene7.com/is/image/dmqualcommprod/Qualcomm_AIHub_image2-1?$QC_Responsive$&fmt=png-alpha" -o /data/local/tmp/qualcomm.png
    cp /data/local/tmp/qualcomm.png /sdcard/Download/
    ```
    The image will appear in the app's image picker when you tap the image button in a VLM chat.
  </Step>
</Steps>

## **Option 2: Build the demo app from source**

Build the demo in Android Studio — useful for modifying the demo UI or pinning a specific published SDK version.

<Note>
Gradle pulls the GenieX SDK from **Maven Central** (coordinates: `com.qualcomm.qti:geniex-android:0.1.6`). You do **not** need to build the native SDK yourself for this path.
</Note>

<Steps>
  <Step title="Clone the repo">
    ```bash
    git clone https://github.com/qcom-ai-hub/geniex.git
    cd geniex
    ```
  </Step>
  <Step title="Open the demo project in Android Studio">
    Open **`examples/android/`** (the demo app). Do **not** open `bindings/android/` — that's the SDK binding module, which this flow consumes from Maven Central.
  </Step>
  <Step title="Sync Gradle">
    Android Studio will prompt you to sync. The sync resolves `com.qualcomm.qti:geniex-android:0.1.6` from Maven Central — this is the only SDK artifact the demo needs.
  </Step>
  <Step title="Connect a device">
    Plug in a Snapdragon 8 Elite or 8 Elite Gen 5 phone with USB debugging enabled (see Option 1a), and let Android Studio detect it.
  </Step>
  <Step title="Run">
    Click **Run ▶**. Android Studio builds the APK, installs it on the phone, and launches the demo.
  </Step>
</Steps>

## **Using the demo**

Once the app launches:

1. **Pick a model** from the dropdown at the top:
   - **Hugging Face GGUF (llama.cpp)** — Qwen3, Phi-4, Gemma, Granite, Qwen3-VL, etc.
   - **AI Hub NPU (QAIRT)** — `qualcomm/Qwen3-4B-Instruct-2507`, `qualcomm/Qwen2.5-VL-7B-Instruct`.
2. **Download** — tap Download. Progress is resumable: killing the app mid-download and relaunching resumes from the last byte.
3. **Load** — a dialog lets you pick the execution target (NPU / GPU / CPU for GGUF; NPU-only for AI Hub models).
4. **Chat** — type text, or tap the image / audio button for VLMs.
