## **Running Your First Model**

<Tabs>
  <Tab title="Windows ARM64">
    ### **Windows ARM64 Installation**

    Download the installer:

    - [GenieX CLI Download](https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-cli.exe)

    click the downloaded `.exe` file and follow the installation. The App is not signed yet, so you need to click "More info" then click "run anyway".
    
    Currently, we support LLM and Multimodal models. Before you run any command, please firstly add geniex to your `PATH`:
    ```powershell windows
    Set-Alias geniex (where.exe geniex)
    ```

    ### **Running Your First Model through QAIRT plugin**

    **Language Model (LLM)**
    Remember to select `Q4_0` as quant type which has the best support with llama.cpp Hexagon backend.

    ```bash Windows
   geniex infer qualcomm/Qwen3-4B
    ```

    **Multimodal Model**
    ```bash Windows
    geniex infer qualcomm/Qwen2.5-VL-7B-Instruct
    ```

    ### **Running Your First Model through llama.cpp plugin**
    **Language Model (LLM)**

    ```bash Windows
   geniex infer unsloth/Qwen3.5-0.8B-GGUF
    ```

    **Multimodal Model**
    ```bash Windows
    geniex infer Qwen/Qwen3-VL-2B-Instruct-GGUF
    ```

    <Tip>To try other GGUF models, visit Hugging Face, copy the path of any compatible GGUF model (e.g., `unsloth/Phi-4-mini-instruct-GGUF`), and replace the model path in the command above.</Tip>
