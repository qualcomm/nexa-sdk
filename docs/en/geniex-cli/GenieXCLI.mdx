## **Model Inference**

### **geniex pull**

Download a model and store it locally.

```powershell
geniex pull <model-name>
```

The CLI guides you through a setup process:

**1. Device selection**

On first run (for both QAIRT and llama.cpp plugins), you will be prompted to select your device:
```powershell
Select your device
┃ > Snapdragon X Elite CRD
┃   Snapdragon X Plus 8-Core CRD
┃   Snapdragon X2 Elite CRD
```

To change your device later:
```powershell
geniex config set device
```

**2. Quantization selection (llama.cpp plugin only)**

For llama.cpp models, the CLI prompts you to pick a quantized version:
```powershell
Choose a quant version to download
> Q4_0       [1.2 GiB] (default)
  Q8_0       [2.0 GiB]
  F16        [3.8 GiB]
```

<Tip>`Q4_0` is recommended for best Hexagon NPU performance.</Tip>

QAIRT plugin models come pre-quantized — no selection needed.


### **geniex infer — LLM**

Launch an interactive chat session with a language model.

```powershell
geniex infer qualcomm/Qwen3-4B
```

**Thinking mode** — control whether the model shows its reasoning before responding:

```powershell
geniex infer qualcomm/Qwen3-4B --think      # show reasoning steps
geniex infer qualcomm/Qwen3-4B --no-think   # respond directly
```

**Device selection** — choose which backend to run on (default: `npu`):

```powershell
# llama.cpp models support all backends
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device npu
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device gpu
geniex infer unsloth/Qwen3.5-0.8B-GGUF --device cpu

# QAIRT models only support NPU
geniex infer qualcomm/Qwen3-4B --device npu
```

<Warning>QAIRT models run on NPU only. Using `--device cpu` or `--device gpu` with a QAIRT model will return an error.</Warning>

### **geniex infer — VLM**

Run vision-language inference with text-only or image input:

```bash
geniex infer Qualcomm/Qwen2.5-VL-7B-Instruct-GGUF
```

If you only want text input, simply launch the command and begin chatting.
To have the model respond from an image, provide the **absolute path** or simply drag the image file into your terminal, for example:

```bash
Describe this picture </full/path/to/image.png>
```


## **Configuration Flags**

These flags can be passed to `geniex infer` to control model loading and generation behavior.

### Sampler Flags

Control how the model selects tokens during generation.

| Flag | Type | Default | Description |
|---|---|---|---|
| `--temperature` | float | — | Sampling temperature. Higher values increase randomness. |
| `--top-p` | float | — | Top-p (nucleus) sampling threshold. |
| `--top-k` | int | — | Top-k sampling. Only consider the top-k most likely tokens. |
| `--min-p` | float | — | Min-p sampling threshold. |
| `--repetition-penalty` | float | `1` | Penalize repeated tokens. Values > 1 reduce repetition. |
| `--presence-penalty` | float | — | Penalize tokens that have appeared at all. |
| `--frequency-penalty` | float | — | Penalize tokens proportional to their frequency. |
| `--seed` | int | — | Random seed for reproducible outputs. |
| `--grammar-path` | string | — | Path to a GBNF grammar file for constrained generation. |
| `--grammar-string` | string | — | Inline grammar in GBNF string format. |
| `--enable-json` | — | — | Force JSON-only output. |

### Model Flags

Control model loading, context, and generation limits.

| Flag | Type | Default | Description |
|---|---|---|---|
| `-n`, `--ngl` | int | `999` | Number of layers to offload to GPU. |
| `--nctx` | int | `4096` | Context window size (max input + output tokens). |
| `--max-tokens` | int | `2048` | Maximum tokens to generate per response. |
| `--stop` | string[] | — | Stop sequences (can be specified multiple times). |
| `--stop-file` | string | — | File containing stop sequences (one per line). |
| `--enable-think` | — | `true` | Enable thinking mode for reasoning models. |
| `--hide-think` | — | — | Hide the model's internal reasoning from output. |
| `-s`, `--system-prompt` | string | — | System prompt to set model behavior. |

## **Utility Commands**

| Command | Description | Example |
|---|---|---|
| `geniex list` | Display all downloaded models with their names and sizes. | `geniex list` |
| `geniex remove <model>` | Remove a specific local model by name. | `geniex remove unsloth/Qwen3-0.6B-GGUF` |
| `geniex clean` | Delete all locally cached models. | `geniex clean` |
| `geniex infer -h` | Show help menu for `geniex infer`. | `geniex infer -h` |
| `geniex serve -h` | Show help menu for `geniex serve`. | `geniex serve -h` |
