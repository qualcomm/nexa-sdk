GenieX is an on-device Gen AI inference runtime built for Qualcomm platforms. It is the easiest way to run frontier language and vision-language models locally on Hexagon NPU, Adreno GPU, or CPU with a few lines of code. It is the community version of Qualcomm GENIE.

## **Architecture**

  src="/Mintlify-image/geniex_arch.png"

GenieX exposes five entry points — CLI, Python, Java/Kotlin, Docker, and an OpenAI-compatible server — over a single SDK that dispatches to either the **llama.cpp runtime** (GGML kernels for CPU / GPU / Hexagon HTP) or the **QAIRT runtime** (Qualcomm AI Runtime, NPU-only). The same SDK runs on Windows ARM64, Android, and Linux ARM64.

## **Why two runtimes?**

So you get both **broad model coverage** and **optimal performance** in one stack:

- **Most models just work** — point GenieX at almost any GGUF on Hugging Face and it runs on CPU / GPU / NPU via llama.cpp.
- **AI Hub models run optimally** — models published to [Qualcomm AI Hub](https://aihub.qualcomm.com/) are pre-compiled per chipset and run through QAIRT on the Hexagon NPU for peak on-device performance.

See [Choose your runtime](/en/get-started/choose-runtime) for when to pick which.

## **What you can do with GenieX**

- **Run models locally** on Snapdragon X (Windows ARM64), Snapdragon 8 Elite (Android), and Dragonwing IoT chipsets.
- **Pick a runtime** — `llama.cpp` for any community GGUF model, `QAIRT` for AI Hub pre-compiled NPU bundles.
- **Build apps** through the CLI, an OpenAI-compatible local server, the Python SDK, the Android SDK, or a Docker image.
