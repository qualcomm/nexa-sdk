## **运行你的第一个模型**

<Tabs>
  <Tab title="Windows ARM64">
    ### **Windows ARM64 安装**

    下载安装程序：

    - [GenieX CLI 下载](https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-cli.exe)

    点击下载的 `.exe` 文件并按照提示完成安装。应用尚未签名，因此需要点击"更多信息"然后点击"仍然运行"。
    
    目前支持 LLM 和多模态模型。运行任何命令前，请先将 geniex 添加到 `PATH`：
    ```powershell windows
    Set-Alias geniex (where.exe geniex)
    ```

    ### **通过 QAIRT 插件运行第一个模型**

    **语言模型（LLM）**

    ```bash Windows
   geniex infer qualcomm/Qwen3-4B
    ```

    **多模态模型**
    ```bash Windows
    geniex infer qualcomm/Qwen2.5-VL-7B-Instruct
    ```

    ### **通过 llama.cpp 插件运行第一个模型**
    请选择 `Q4_0` 作为量化类型，这是 llama.cpp Hexagon 后端支持最好的格式。

    **语言模型（LLM）**

    ```bash Windows
    geniex infer unsloth/Qwen3.5-0.8B-GGUF
    ```

    **多模态模型**
    ```bash Windows
    geniex infer Qwen/Qwen3-VL-2B-Instruct-GGUF
    ```

    系统提示时，请选择：
    - **模型类型**：视觉语言模型选择 `vlm`，语言模型选择 `llm`。目前 Gemma4 和 Qwen3.5 请选择 `llm`，`vlm` 路径暂不支持。

    - **量化**：`Q4_0` — llama.cpp 的 Hexagon NPU 支持最佳

    <Note>对 `Qwen3.5` 和 `Gemma4` 的多模态支持即将推出。</Note>

    <Tip>如需尝试其他 GGUF 模型，请访问 Hugging Face，复制任意兼容 GGUF 模型的路径（例如 `unsloth/Phi-4-mini-instruct-GGUF`），替换上述命令中的模型路径即可。</Tip>
