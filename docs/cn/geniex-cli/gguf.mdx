llama.cpp 插件支持社区中的**任意 GGUF 模型** — 你不限于以下列表。以下是经过测试的示例，在骁龙 NPU 上使用 `Q4_0` 量化表现良好。

## **示例模型**

| 模型 ID |
|---|
| unsloth/Qwen3.5-0.8B-GGUF |
| unsloth/Qwen3.5-2B-GGUF |
| unsloth/Gemma4-E4B-it-GGUF |
| unsloth/Gemma4-E2B-it-GGUF |
| unsloth/Qwen3-0.6B-GGUF |
| unsloth/Qwen3-1.7B-GGUF |
| unsloth/Ministral-3-3B-Instruct-2507-GGUF |
| unsloth/GPT-OSS-20B-GGUF |
| ibm-granite/granite-4.0-micro-GGUF |
| unsloth/Phi-4-mini-instruct-GGUF |
| Qwen/Qwen3-VL-2B-Instruct-GGUF |
| Qwen/Qwen3-VL-4B-Instruct-GGUF |


运行模型，以 `unsloth/Qwen3.5-0.8B-GGUF` 为例：
```powershell windows
geniex infer unsloth/Qwen3.5-0.8B-GGUF
```

系统提示时，请选择：
- **模型类型**：视觉语言模型（Qwen3-VL）选择 `vlm`，其他选择 `llm`
- **量化**：`Q4_0` — llama.cpp 的 Hexagon NPU 支持最佳

<Note>对 `Qwen3.5` 和 `Gemma4` 的多模态支持即将推出。</Note>
