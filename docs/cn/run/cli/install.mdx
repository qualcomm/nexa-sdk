CLI 提供 Windows ARM64 原生可执行文件，以及面向 Dragonwing IoT 等设备的 Linux ARM64 Docker 镜像。

<Tabs>
  <Tab title="Windows ARM64">
    ### **前置条件**

    - 骁龙 X 系列设备（X Elite 或 X2 Elite）。详见[支持的设备](/cn/get-started/devices)。
    - PowerShell 或任意 Windows ARM64 上可用的 shell。

    ### **下载**

    <Card title="GenieX CLI 安装包" icon="download" href="https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/geniex-cli.exe">
      最新 Windows ARM64 GenieX CLI。点击下载好的 `.exe` 并按提示完成安装。
    </Card>

    安装包尚未代码签名——Windows SmartScreen 会发出告警。点击 **更多信息 → 仍要运行**。

    ### **加入 PATH**

    让 `geniex` 在 shell 中可用：

    ```powershell windows
    Set-Alias geniex (where.exe geniex)
    ```

    ### **验证**

    ```powershell windows
    geniex --help
    ```

    现在跳转到[快速入门](/cn/run/cli/quickstart)。
  </Tab>

  <Tab title="Linux ARM64 (Native)">
    直接安装到骁龙 ARM64 Linux 主机（EVK、容器或任何带有高通 BSP 的设备）——无需 Docker。

    ### **前置条件**

    - Linux ARM64 主机。
    - 高通 IoT 设备——详见[支持的设备](/cn/get-started/devices#iot-linux-arm64)。

    ### **安装**

    ```bash bash
    curl -fsSL https://qaihub-public-assets.s3.us-west-2.amazonaws.com/qai-hub-geniex/install.sh | sh
    ```

    脚本会下载最新稳定版本、校验 SHA256 并原子化安装，全程**无需 `sudo`**：

    - **root**（EVK / 容器）→ 二进制位于 `/usr/local/lib/geniex/`，启动器位于 `/usr/local/bin/geniex`。
    - **非 root** → 二进制位于 `${XDG_DATA_HOME:-~/.local/share}/geniex/`，启动器位于 `~/.local/bin/geniex`。

    常用参数（写在 `sh -s --` 后）：

    ```bash bash
    # 锁定指定版本
    curl -fsSL .../install.sh | sh -s -- --version v0.1.8

    # 自定义安装路径
    curl -fsSL .../install.sh | sh -s -- --prefix /opt/geniex
    ```

    ### **加入 PATH**

    如果启动器目录不在 `PATH` 中，安装脚本会打印需要追加的命令行——一般为：

    ```bash bash
    echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bashrc   # 或 ~/.zshrc / ~/.profile
    ```

    打开新 shell 或 `source` 上述文件。

    ### **验证**

    ```bash bash
    geniex --help
    ```

  </Tab>

  <Tab title="Linux ARM64 (Docker)">
    Linux 路径通过 Linux ARM64 容器镜像运行，并支持 NPU 访问（适用于 Dragonwing IQ9 等 IoT 平台）。

    ### **前置条件**

    - 安装了 Docker 的 Linux ARM64 主机。
    - 高通 IoT 设备——详见[支持的设备](/cn/get-started/devices#iot-linux-arm64)。
    - 拥有 GHCR 读权限的 GitHub Personal Access Token (PAT)。

    ### **拉取镜像**

    登录 GitHub Container Registry 后再拉取：

    ```bash bash
    export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxx
    docker login ghcr.io -u <your-github-username> -p "$GITHUB_TOKEN"

    docker pull ghcr.io/qcom-ai-hub/geniex-cli:latest
    ```

    ### **交互式运行**

    进入容器内交互式 shell：

    ```bash bash
    docker run -it --rm --privileged \
      -v "$PWD/data:/data" \
      -v /usr/lib:/opt/qcom-lib:ro \
      ghcr.io/qcom-ai-hub/geniex-cli:latest
    ```

    ### **验证**

    在容器 shell 中运行测试命令：

    ```bash bash
    geniex --help
    ```

    <Note>NPU 访问需要 `--privileged` 标志。</Note>

    服务器模式参见[本地服务器](/cn/run/cli/local-server)。
  </Tab>
</Tabs>
