示例应用将模型分为两类：

- **GGUF via `llama_cpp`** — 在 CPU / Adreno GPU / Hexagon NPU 上运行。加载时选择设备。
- **AI Hub via `qairt`** — 来自 Qualcomm AI Hub 的预编译 NPU 专用包。在骁龙 8 Elite（芯片组 `SM8750`）和骁龙 8 Elite Gen 5（芯片组 `SM8850`）上运行。

以下两个示例各展示一种。

## **1. GGUF LLM：`unsloth/Qwen3-0.6B-GGUF`**

小型 Qwen3 0.6B 聊天模型，Q4_0 量化。下载快，可在任何设备上运行。

<Steps>
  <Step title="安装应用">
    参见快速入门 — [安装预构建 APK](/cn/geniex-android/quickstart#方式一安装预构建-apk) 或在 Android Studio 中[从源码构建](/cn/geniex-android/quickstart#方式二从源码构建示例应用)。
  </Step>
  <Step title="选择模型">
    打开顶部模型下拉菜单，选择 **Qwen3-0.6B (GGUF)**。
  </Step>
  <Step title="下载">
    点击 **Download**。进度由 Rust 模型管理器实时流式传输；下载中途关闭应用后重启会从断点继续。
  </Step>
  <Step title="加载">
    点击 **Load**。弹出对话框 — 选择 **NPU**、**GPU** 或 **CPU**：
    - **NPU**（骁龙推荐）— Hexagon NPU 加速。
    - **GPU** — 通过 OpenCL 使用 Adreno。
    - **CPU** — 适用于任何 ARM64 设备。
  </Step>
  <Step title="聊天">
    在输入栏输入消息，点击 **Send**。点击 **Clear** 或卸载模型以重置对话。
  </Step>
</Steps>

## **2. AI Hub VLM：`qualcomm/Qwen2.5-VL-7B-Instruct`**

为骁龙 8 Elite 和 8 Elite Gen 5 预编译的视觉语言模型。

<Warning>
需要芯片组 `SM8750`（骁龙 8 Elite）或 `SM8850`（骁龙 8 Elite Gen 5）的设备。
</Warning>

<Steps>
  <Step title="选择模型">
    打开下拉菜单，选择 **Qwen2.5-VL-7B-Instruct (NPU)**。
  </Step>
  <Step title="下载">
    点击 **Download**。AI Hub 包比 GGUF 模型更大（约 4.6 GB）— 请保持 Wi-Fi 连接。下载支持跨应用重启断点续传。
  </Step>
  <Step title="加载">
    点击 **Load**。QAIRT 模型的插件选择器仅显示 **NPU**（`qairt` 设计上仅支持 NPU）。确认。
  </Step>
  <Step title="图像聊天">
    点击 **图像** 按钮，从相册选择照片，输入问题如"这张图片里有什么？"，然后发送。VLM 聊天模板由 SDK 自动应用。
  </Step>
</Steps>

## **故障排除**

**"模型加载但生成自重复或无输出"**
确保示例应用将经过聊天模板处理的提示（`applyChatTemplate().formattedText`）传入 `generateStreamFlow`，而非原始用户文本。QAIRT 管线将输入视为已模板化 — 发送原始文本会使模型进入补全模式。

**"AI Hub 拉取失败：`INVALID_INPUT`"**
Android 上 AI Hub 拉取需要显式 `chipset` — 自动检测仅在 Windows-on-Snapdragon 上生效。示例应用的 `model_list.json` 固定为 `SM8750`；如果你自定义了它，请确保该字段存在。

**"QAIRT 加载失败：unknown model name"**
AI Hub 返回的模型 id 必须匹配 QAIRT 插件注册表中的条目（`qwen3_4b_instruct_2507`、`qwen2_5_vl_7b_instruct` 等）。如果你要添加新的 AI Hub 模型，请先在 C++ 端注册 — 参见 `third-party/geniex-qairt/models/{llm,vlm}_model_registry.h`。
