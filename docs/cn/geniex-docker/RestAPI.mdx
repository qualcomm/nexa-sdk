## **快速开始**

使用 API 前，先以服务器模式启动 GenieX SDK Docker 容器：

```bash bash
docker run -it --rm --privileged \
  -v "$PWD/data:/data" \
  -v /usr/lib:/opt/qcom-lib:ro \
  --network=host \
  ghcr.io/qcom-ai-hub/geniex-cli:latest \
  serve
```
服务器默认运行在 `http://127.0.0.1:18181`。保持容器运行，从另一个终端或应用发送请求。

<Info>
你也可以访问交互式 Swagger UI 文档 [`http://127.0.0.1:18181/docs/ui`](http://127.0.0.1:18181/docs/ui)，直接在浏览器中浏览和测试 API 端点。完整可配置选项请参阅[快速入门指南](/cn/geniex-docker/quickstart)。
</Info>

<Note>
`--privileged` 标志是 NPU 访问所必需的。`geniex serve` 不会自动下载模型，请确保预先下载要使用的模型。
</Note>

## 模型选择

GenieX SDK Docker 支持 Linux ARM64 架构。完整的支持模型列表及 Hugging Face 链接，请查看[快速入门指南](/cn/geniex-docker/quickstart#支持的模型)。

## **API 端点**

GenieX SDK REST API 提供 OpenAI 兼容端点，用于各种 AI 任务。详细 API 文档（包括请求/响应格式、示例和所有可用端点），请参阅 [CLI REST API 文档](/cn/geniex-cli/GenieXAPI)。

### **可用端点**

- **`/v1/chat/completions`** - 为对话创建模型响应（LLM 和 VLM）
- **`/v1/models`** - 列出可用模型
- **`/v1/models/{model}`** - 获取指定模型的信息

<Info>
所有 API 端点、请求/响应格式和使用示例均记录在 [CLI REST API 页面](/cn/geniex-cli/GenieXAPI)。无论通过 CLI 还是 Docker 运行，API 接口完全相同——仅服务器启动方式不同。
</Info>

## **Python 示例（OpenAI SDK）**

由于服务器使用 OpenAI 协议，你可以直接将官方 `openai` Python 客户端指向本地端点，复用已有的 OpenAI 代码。先通过 `pip install openai` 安装 SDK，然后：

```python python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:18181/v1",
    api_key="geniex",  # 任意非空字符串，服务器不会校验
)

stream = client.chat.completions.create(
    model="Qwen/Qwen3-0.6B-GGUF:Q4_0",  # org/repo[:quant]，Q4_0 对 Hexagon NPU 支持最佳
    messages=[
        {"role": "user", "content": "你好！请简单介绍一下你自己。"},
    ],
    max_tokens=256,
    temperature=0.7,
    stream=True,
)

for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
print()
```

<Note>
请将 `model` 替换为已经下载到容器 `/data` 卷中的模型。`geniex serve` 不会自动下载模型。可选的 `:<quant>` 后缀用于选择量化版本（例如 `Q4_0`、`Q4_K_M`、`Q8_0`），其中 `Q4_0` 对 llama.cpp Hexagon NPU 后端支持最好。
</Note>
